Using Actions

Introduction

Actions are templates for Data Definition Language (DDL) commands. They are used by the Common Format Designer to generate the scripts implementing a data model into a data server or synchronizing the differences between a data model described in Oracle Data Integrator and its implementation in the data server.

An action corresponds to a DDL operation (create table, drop reference, etc). Each action contains several Action Lines, corresponding to the commands required to perform the DDL operation (for example, dropping a table requires to drop all its constraints first).

Actions are organized into Action Groups. An action group corresponds to a given syntax and/or purpose for the DDL commands (Oracle, SQL-92, etc).

Generic Actions vs. Technology Actions

Actions in Topology Manager may be either attached to a technology or created in the Generic Actions view.

Generic actions correspond to the default syntax. They are compatible with databases supporting the SQL-92 syntax. They are used for technologies without a specific action group, or when a given action is missing from the action group being used.

Creating Actions

To create actions you must first create an action group.

To create an action group for a technology:

  1. Expand the technology, then click the Action Groups node. Right-click, then select Insert Action Group.

  2. Fill in the Name, Code and Description for your action group.

  3. Check the Default Group box if you want this action group to be selected for new models created for this technology. If no default action group is selected for a given technology, new models use the generic actions .

  4. Click OK to save the action group.

To create an action in an action group:

  1. Select the action group, right-click then select Insert Action.

  2. Fill in the Name and Description fields, then select the action Type.

  3. Go to the Details tab.

  4. Select the Add icon to add an action line.
    You may also use the Delete icon to delete invalid action lines, or the Up and Down buttons to reorganize them.

  5. In the Action Line window that appears, fill in the following fields:

    • Name: Name of the action line. This name will be given to the procedure command generated.

    • Ignore Errors: This option must be checked if you do not want the procedure to stop if this specific command returns an error. (For example, dropping a non-existing table should not block the whole process)

    • Commit: Indicates if a commit should be issued at the end of the command.

    • Isolation Level: It is the transaction isolation level for the command.

    • Action Text: Text of the command to execute. You may call the expression editor by clicking .

  6. Click OK to save the action line.

  7. Repeat operations 4 to 6 for each action line you want to create.

  8. Click OK to save the action.

Note: Action lines must be coded using the appropriate Substitution methods. Please refer to the topic Using Substitution Methods in Actions in the Substitution Methods Reference guide for more information.

Note: To create a new actions or groups, it is recommended that you copy an existing action or group and modify it for your requirements. You can perform this operation simply by drag and dropping the object while keeping the CTRL key pressed.

Updating Action Groups

Similarly to knowledge modules, action groups have an import-replace feature to manage updates.

To import an Action Group in replace mode:

  1. Select the Action Group you wish to replace

  2. Right-click on the Action Group, and select Import Replace

  3. Specify the export file.

  4. Click OK

The action group is replaced by the new one. Note that the DDL scripts should be re-generated with the updated group.

Converting Actions to DDL Script

DDL scripts are generated from the actions, based on the differences selected while configuring the DDL script generation. The DDL script is generated in the form of an Oracle Data Integrator procedure with each procedure line corresponding to an action line. Actions lines are sequenced in the procedure using an algorithm ensuring that the selected changes will be performed while avoiding useless command repetitions.

Ordering Actions in DDL Scripts

Actions usually appear in the generated script in the following sequence:

  1. Begin action
  2. Drop actions
  3. Alteration actions: Modify, Enable, Disable, Rename
  4. Add (and Create) actions
  5. <unknown> actions
  6. End actions
