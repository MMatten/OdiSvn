Session

A session is an execution (of a scenario, an interface, a package or a procedure, ...) undertaken by an execution agent. A session is made up of steps which are made up of tasks.

Definition

Properties

Description

Name

Name of the scenario, the package, the interface, the procedure, etc, executed in this session.

Version

Version (for a scenario).

Execution Context

The context in which the execution was run.

Execution Agent

Agent responsible for execution of the tasks in this session.

State

The session state when the session window was opened. The possible states are Done, Error, Running, Waiting and Warning.

Stop

Stops the Session

Restart

Restarts the session

Execution

Execution information details

Start

Start date and time of execution of the session.

End

End date and time of execution of the scenario.

Duration

The time taken for execution of the scenario.

Return code

Return code for the session

Message

Session execution error message, if needed.
