Upgrading the master repository

To upgrade the master repository:

  1. Make a backup of your master repository, using the Database's backup tools.

  2. In the Start Menu, select Programs > Oracle Data Integrator > Repository Management > Master Repository Upgrade, or Launch bin/mupgrade.bat or bin/mupgrade.sh.

  3. Select the Login corresponding to your Master Repository connection, or complete the fields:

    • Driver: the driver used to access the technology which hosts the repository. For more information, refer to the JDBC URL Sample section.

    • URL: The complete path for the data server hosting the repository. For more information, refer to the JDBC URL Sample section.

    • User: The user id / login of the owner of the tables

    • Password: This user's password.

    • Technologies: Select the technology your repository will be based on from the list.

    • Language: Select your master repository's language.

  4. Click OK.

Repository upgrade begins. You can follow the procedure on the console. A popup appears when the master repository is upgraded.

Once the master repository is upgraded, it is advised to also upgrade the work repositories attached to this work repository.

Important! The upgrade of a master and work repository performs modifications in the repository's structure. This sequence of modifications cannot be undone and is indivisible. If an error occurs during a repository upgrade process, you must restore the repository database backup before restarting the repository upgrade process.
