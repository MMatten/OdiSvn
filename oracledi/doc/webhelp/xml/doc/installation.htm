Installation

Note for Data Integrator Users : If you are using Oracle Data Integrator Driver for XML with Oracle Data Integrator, it is already installed and configured by the Data Integrator Setup. The process below is provided for other application using this driver.

Prerequisites

To use OracleDI Driver for XML, you need a Java Virtual Machine 1.2.2 or higher. To check your Java Machine version, type the java -version command in a command prompt.

If needed, you can download the appropriate Java Machine at Sun Microsystem's website.

Installing the Package

Unbundling The Downloaded File

You can unbundle the downloaded file using a tool such as WinZip, Pkunzip or jar. For example:

c:> jar xvf OracleDIXml.zip

or

c:> pkunzip -d OracleDIXml.zip

Note: If you are using a tool that does not preserve path names by default, be sure to specify the option that preserves path names. For example, specify -d for pkunzip

The downloaded file contains the following files:

File/Directory Name

Description

/drivers/snpsxmlo.jar

Oracle Data Integrator Driver for XML

/drivers/crimson.jar

Crimson XML Java Parser

/doc

OracleDI Driver for XML Documentation

/demo

Contains the demo and sample files for OracleDI Driver for XML

Configuring the Classpath

Once you have the driver installed, you can use the driver simply by adding the jar and zip files into your Java classpath.  For example:

Under Windows:

set CLASSPATH=%CLASSPATH%;c:\snpsxmlo.jar;c:\crimson.jar

Under Unix:

setenv CLASSPATH "/usr/local/snpsxmlo.jar:/usr/local/crimson.jar:$CLASSPATH"

You can also specify the jar and zip files on the command line when running applications. For example:

java -cp .;c:\snpsxmlo.jar;c:\crimson.jar myApplication

Important Note: If using an External Database engine, you must also make sure that the JDBC driver used to connect the external database, as well as the .properties file are in the classpath.

Important Note:If using XSD validation, you must replace the Crimson (crimson.jar) parser by the Xerces parser (xerces.jar) and modify the application classpath as a consequence.
