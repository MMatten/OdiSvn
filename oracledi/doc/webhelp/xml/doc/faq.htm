Questions

How do I add the driver in the classpath?

Add the snpsxmlo.jar and crimson.jar files to your classpath or copy them in your application's default directory. Please ensure also that the license key file (jdbcxkey.xml) is in the same directory as the driver file. If using XSD validation, you should use xerces.jar instead of crimson.jar.

Which Java version is required?

The driver OracleDI Driver for XML is a JDBC 2.0 driver and supports Java Virtual Machine 1.2.2 or higher.

In which operating system environments does your JDBC driver work?

OracleDI Driver for XML is a type 4 driver. Type 4 drivers work on all operating systems with a Java Virtual Machine.

I downloaded the latest version of the driver. Do I need a new license key file (jdbcxkey.xml)?

You do not need a license key file for this release of the driver.

What are the JDBC API features implemented in this driver?

The JDBC implemented features are detailed in this appendix.

Are namespaces supported?

Yes. The JDBC driver for XML recognizes XML namespaces present within an XML file, and is able to reconstruct them correctly when synchronizing the file.

Is XSD supported?

XSD is supported in OracleDI Driver for XML for describing XML file structures. See XML Schema Supported Features for more information.
In addition, OracleDI Driver for XML supports document validation against XSD schemas specified within the XML file.

Troubleshooting

java.sql.SQLException: No suitable driver

This error message means that no driver able to process the URL is registered. 2 causes:

  • The JDBC URL is incorrect. Check that the URL syntax is valid. See Driver Configuration.

  • The driver is not in the classpath. See "How to add the driver to the classpath?".

Exception: java.lang.ClassNotFoundException

The classes are not in your classpath. Please see "How to add the driver to the classpath?".

IndexOutOfRange exception when naming the xml driver schema

 It is not possible to have a schema named like one of the generated tables. Please avoid naming the schema like one of the elements of the XML structure.

java.sql.SQLException: General error java.lang.NullPointerException in statement [select xxxx from xxxx]  

This error message can have various causes:

  • Check if the table name is valid
  • Set the current schema where the table is loaded (ex: SET SCHEMA TheTableSchema)
  • Or prefix the table name with the schema name where the table is loaded if you do not wish to change your current schema (ex: select xxx from TheTableSchema.xxx)
java.lang.NullPointerException when synchronizing XML files from database

Some of the schemas that are being synchronized were created in read only mode. Synchronizing the file from the schema is impossible.

java.sql.SQLException: The DTD file "xxxxxxx.dtd" doesn't exist

This exception may occurs when trying to load an XML file. The error message has two causes:

  • The path of the DTD file is incorrect.

  • The corresponding XML file was already opened by another schema (during connection for instance). 

java.sql.SQLException: File xxxxxxx.xml is already locked by another instance of the XML driver

This exception may occurs when trying to load or connect to an XML file. The error message can have two causes:

  • The file that you are trying to load has already been loaded by another instance of the driver.

  • Previous connections to this file were not properly closed. The XML files were not unlocked. Delete the lock files and try again.
