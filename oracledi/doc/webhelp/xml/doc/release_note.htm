Oracle Data Integrator Driver for XML Release Notes

Changes in version 10.1.3.4

New Features

  • Full support for Boolean types. Boolean values are mapped to a numeric column containing either 0 or 1. Entries in XML and XSD files may also be "true" or "false". See XML Schema Supported Features.

  • Prefixes in the generated tables are now more consistent with namespaces.

  • Changing namespace prefix definitions within an XSD file is now supported.

  • Changing the target namespace in an XSD file is now supported.

  • Elements can be defined using namespace prefixes to override de current target namespace.

  • Target file can be generated with no default namespace using the no_default_ns URL parameter or the NO_DEFAULT_NS option of the CREATE FILE command.

  • 6353068: Smart namespace prefix generation using the ns_prefix_generation URL parameter.

Changes in version 10.1.3.0

New Features

  • 129950: Improved support for XSD: Element groups are now supported. See XML Schema Supported Features.

Problems Fixed

  • Problem with attribute groups when defined in an included XSD file.
Changes in version 3.03

Problems Fixed

  • 130456: The reverse of a Model based on XSD file does not work

  • Support for the "targetNamespace" attribute.

  • Improved Support for namespace prefixes.

  • 131218: The root element URL parameter is re-activated in order to specify the root element of a DTD or XSD file. This parameter can be used for reverse-engineer a specific message definition from a WSDL file.

  • Elements containing empty PCDATA are allowed even if the element is marked as mandatory.

  • New parameter "numeric_scale" or "ns" to indicate in the URL the scale of the numeric columns generated during the XML to SQL mapping.

  • Long integer values now work correctly.

Problems Fixed

Changes in version 2.18

New Features

  • New "variable_length" and "numeric_length" external database properties to customize default size for the columns created.

  • External database properties can be overridden or set in the URL. They should be prefixed with "dp_"

Problems Fixed

  • "nobu" parameter does not work as expected.

  • "ClassCastException" error appears when accessing a file using XSD type Integer.

  • 128290: A 'java.sql.SQLException: ORDER BY item should be in the SELECT DISTINCT' error occurs when using the DISTINCT keyword in a SELECT statement.

  • Columns corresponding to optional PCDATA elements are reverse-engineered as NOT NULL from an XSD file.

  • Carriage return in attributes and PCDATA are now correctly written to the XML file.

Changes in version 2.11

New Features

  • Support for the xsi:nil="true" attribute in XML elements. Elements with this attribute are considered as containing no PCDATA. If such an element is read from an XML file with no other attribute, it is not generated when synchronizing the XML schema to file.

  • Support for the XML files containing only the root element.

  • Support for XML namespaces.

  • XSD validation support through the VALIDATE command or the ValidateXMLFile class.

  • Existing XSD information is now correctly read/reconstructed.

Fixes

  • 123305: DTD files generated by the driver have all elements marked as REQUIRED. This problem is now solved.

Note: This version has a different specification than version 1.x concerning column names. When in the XML to SQL mapping a column has the same name than the table containing it, it will be named COL_<column_name> (in v1, they were named <column_name>_DATA). Besides, object names that are in the reserved keyword list are followed by an underscore (In v1.x, they where prefixed by the underscore).
