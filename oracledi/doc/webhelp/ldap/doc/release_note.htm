Oracle Data Integrator Driver for LDAP Release Notes

Changes in version 10.1.3

Fixes

  • Fixed bug with special characters such as "/" in the DN.

Changes in version 1.0l

New Features

  • New "jdbc:snps:ldap2" URL syntax for non-encrypted passwords.

Fixes

  • Special characters such as "/" are now supported in the DN.

  • 123183: New reserved keyword: "Unique".

  • 123978: When performing a reverse-engineering, decimal columns precision is set to -1.

Changes in version 1.0k

Features

  • Support for LDAPS (LDAP over SSL) protocol. The LDAPS URL must start with ldaps://. To connect a server using LDAPS, you must manually install the certificate in the java machine. See the keytool program provided with the JVM for more information.

Changes in version 1.0j

Features

  • New reserved keywords: "uid", "description".

  • Tracing improvement..

Fixes

  • Fixed: Errors such as "bad column count" or "no such column" could appear while inserting data into an LDAP tree.

  • Fixed: Insert and Update operation causing errors at LDAP-level are not correctly raised.

  • Fixed: During update operations, only one attribute is effectively updated.

  • Fixed: Reserved keywords for an external database are ignored.

  • "Null Pointer Exception" error no longer appears when updating an element with null values.

  • It is now mandatory to encrypt passwords into the External Database properties file.

  • Fixed: Unexpected behavior if using case-sensitivity.

Changes in version 1.0f

Features

  • Advanced logging system. See log_level URL parameter description for more information.

Fixes

  • 122253: Password could be recognized non encrypted in the LDAP bundle file. This option is no longer possible.

Changes in version 1.0e

Features

  • 120468: In addition to the driver installation directory, the license file is also searched in the classpath.

  • 120870: The license file can be inserted into a .jar file.

  • 120871: New reserved word: INSERT.

  • 122020: Non alphanumeric characters ([a-z], [A-Z], [0-9]) in LDAP classes and attributes names are now converted to underscore ("_")  in the LDAP to relational mapping. Exception: If non alphanumeric, the first character is converted to "x".

Fixes

  • 120111: Displaying LDAP data in Data Integrator only shows the 25 first lines of the datastore.
