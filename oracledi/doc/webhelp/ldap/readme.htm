Product Description

Information describing the various users, applications, files, printers, and other resources accessible from a network is often collected into a special database called an LDAP directory. Accessing this information often requires specific coding which can be complex, expensive and time and consuming. In order to simplify information retrieval from this type of data source Oracle Data Integrator includes a driver based on the JDBC API.

The Oracle Data Integrator Driver for LDAP is the world's first type-4 (all Java) JDBC driver for LDAP directories. Unlike other middleware products which transfer data between an LDAP directory and other directory-enabled applications and databases, OracleDI Driver for LDAP treats the directory as a JDBC data source. This allows other JDBC applications to use LDAP directories as SQL-compliant data sources.

With the Oracle Data Integrator Driver for LDAP you can query LDAP trees using standard SQL syntax and also be able to persist modifications to the resource classes and their attributes.

Features & Benefits

LDAP is widely used as a common protocol for accessing directory service applications. With OracleDI Driver for LDAP your Java applications will be able to manipulate complex LDAP trees using standard ISO SQL syntax. You will be able to query your LDAP directories using ISO SQL commands and features (joins, select, update, insert, delete) and also be able to control the persistence of directory updates.

OracleDI Driver for LDAP supports:

  • Manipulation of  LDAP entries and their object classes and attributes
  • Standard SQL (Structured Query Language) Syntax
  • Correlated subqueries, inner and outer joins
  • ORDER BY and GROUP BY
  • COUNT, SUM, MIN, MAX, AVG and other functions
  • All Standard SQL functions
  • Referential Integrity (foreign keys)
  • Persisting modifications on data in directories

With OracleDI Driver for LDAP you will be able to:

  • have easy and rapid access to your LDAP trees,
  • access your resource directories as if they are relational databases,
  • avoid unnecessary training for LDAP search and filter languages.
