The Scenario Launcher is a Java applet allowing you to launch a Sunopsis Scenario from a Web interface on a running agent.

Using the Applet

Prerequisites:

Server

You must install the contents of the /sunopsis/lib/web directory in your web server, or launch the Sunopsis Agent's internal web server. For more information about launching the agent, refer to the Sunopsis Documentation.

Client

Java-enabled browser.

Launching a Scenario

  1. Specify :
  • The port where the Physical Agent is installed and launched (command agent.bat or agent.sh). If using the Sunopsis Agent internal webserver, the port is automatically set.

    Note : Java applets can only connect to the host which have served them. Therefore the host is automatically set to the name of the http server on which the Scenario Launcher is installed. The agent must be launched on the web server machine. 

  • A valid Sunopsis User and password. This user must have the right on the execution Context
  • The execution mode: Synchronous execution forces the applet to wait for the end of the execution and the final return code.
  • The variables and the values associated to the variables.
  1. Click on Execute
  2. The execution result appears in the field.

For more information about the parameters, please refer to the Sunopsis Documentation (Topic "Launching a scenario from an OS command").

Using the Applet in your webpages

Using your web server

You may use the applet in your webpages, using the following syntax : 

<applet 
width="350" height="500" 
code="com.sunopsis.remote.applet.SnpsStartScenApplet"
archive="snpsweb.jar">
If this text appears then the applet interpreter does 
not work correctly.
<param name="SNPS_AGENT_PORT" value="20910"/>

The applet parameter SNPS_AGENT_PORT allows you to preset the port on which the launched agent is listening.

Using the Sunopsis Agent internal web server

When launching the agent with the internal webserver activated, the default page used is index.snp, instead of index.html.
This page is parsed for the string #SNPS_AGENT_PORT, which is used to set the SNPS_AGENT_PORT applet parameter. The string #SNPS_AGENT_PORT is replaced by the port on which the current agent is listening.

This is why the file index.snp is similar to index.htm, but the applet is inserted in the page using the syntax :

<applet 
width="350" height="500" 
code="com.sunopsis.remote.applet.SnpsStartScenApplet"
archive="snpsweb.jar">
If this text appears then the applet interpreter does 
not work correctly.
<param name="SNPS_AGENT_PORT" value="#SNPS_AGENT_PORT"/>
